Ejercicio 1

¿Qué significan los conceptos de scope, hoisting y mutabilidad en JavaScript? Explicá con ejemplos simples.

Scope: Es el alcance de una variable. Por ejemplo, var tiene alcance global o de función, mientras que let y const tienen alcance de bloque.

Hoisting: Es cuando las variables o funciones se "elevan" al inicio de su ámbito. Las funciones se elevan completas. Las variables con var se elevan pero se inicializan con undefined. Las variables con let y const también se elevan pero quedan en una "zona muerta" hasta su declaración.

Mutabilidad: Es la capacidad de cambiar el valor de una variable después de crearla. Los datos mutables pueden modificarse. Los datos inmutables no pueden cambiarse, como los valores primitivos.

Ejercicio 2

¿Qué diferencias hay entre var, let y const en cuanto a scope, hoisting y mutabilidad? ¿Cuándo usarías uno y cuándo otro?

var: Scope global o de función. Hoisting con undefined. Mutable. No se recomienda usar.

let: Scope de bloque. Hoisting pero en zona muerta. Mutable. Se usa cuando el valor va a cambiar.

const: Scope de bloque. Hoisting pero en zona muerta. Inmutable (no se puede reasignar). Se usa cuando el valor no va a cambiar.

Ejercicio 3

¿Qué pasa si intentás usar una variable declarada con let antes de su declaración? ¿Y con var?

Con let da un error de referencia porque está en la "zona muerta temporal". Con var devuelve undefined porque se eleva pero no se inicializa.

Ejercicio 4

¿Qué significa que JavaScript sea un lenguaje de tipado dinámico?

Significa que una variable puede cambiar de tipo en tiempo de ejecución. Una misma variable puede contener primero un string, luego un número, después un array, etc. No es necesario declarar el tipo al crear la variable.

Ejercicio 5

¿Cuál es la diferencia entre undefined y null en JavaScript?

undefined significa que una variable fue declarada pero no tiene valor asignado. null es un valor que se asigna intencionalmente para indicar "vacío" o "sin valor". Es una diferencia semántica: undefined es falta de valor, null es valor vacío.

Ejercicio 6

¿Qué tipo de valor es NaN y en qué situaciones puede aparecer?

NaN (Not a Number) es de tipo number. Puede aparecer en:

  • Operaciones matemáticas inválidas: 0 / 0, raíz cuadrada de un número negativo
  • Conversiones fallidas: parseInt("texto")
  • Operaciones con valores indefinidos
  • Cualquier operación que involucre un NaN también devuelve NaN

Ejercicio 7

¿Qué hace el operador typeof y qué valores puede devolver?

typeof devuelve el tipo de dato de una variable. Puede devolver: "string", "number", "boolean", "undefined", "object", "function", "bigint", "symbol".

Ejercicio 8

¿Qué diferencias hay entre usar comillas simples, dobles o backticks para strings?

En la práctica, no hay diferencias funcionales. Las tres sirven para crear strings. Los backticks (`` ` ``) permiten interpolación de variables con ${variable} y strings multilínea. Las comillas simples y dobles son equivalentes, solo se diferencian en que una permite incluir la otra sin escaparla.

Ejercicio 9

¿Por qué typeof null devuelve "object" y por qué se considera un bug histórico?

typeof null devuelve "object" porque en las primeras versiones de JavaScript, los valores se representaban con etiquetas de tipo. null tenía la etiqueta de objeto (0x00) y esto quedó así por compatibilidad. Se considera un bug histórico que no se puede corregir porque rompería mucho código existente.
