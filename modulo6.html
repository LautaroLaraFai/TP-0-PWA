Ejercicio 1

¿Qué es una función de orden superior (higher-order function)?

Es una función que puede tomar una o más funciones como argumentos (callbacks) o devolver una función como resultado. Ejemplos comunes son map(), filter(), reduce().

Ejemplo
function operar(a, b, callback) {
return callback(a, b);
}
const suma = operar(5, 3, (x, y) => x + y); // 8

Ejercicio 2

¿Qué es un callback y para qué se usa?

Un callback es una función que se pasa como argumento a otra función y se ejecuta después de que esta última termine. Se usa principalmente para manejar operaciones asíncronas (eventos, setTimeout, peticiones a servidores) y para personalizar el comportamiento de funciones de orden superior.

Ejemplo
setTimeout(() => {
console.log("Esto se ejecuta después de 1 segundo");
}, 1000);

Ejercicio 3

¿Cuál es la diferencia entre una función normal y una arrow function?

  • Sintaxis: Las arrow functions son más concisas: () => {} vs function() {}
  • this: Las arrow functions no tienen su propio this. Heredan el this del contexto donde fueron creadas (this léxico).
  • Constructor: Las arrow functions no pueden usarse con new (no son constructores).
  • arguments: Las arrow functions no tienen su propio objeto arguments.
Ejemplo
// Función normal
function normal() { console.log(this); }
// Arrow function
const arrow = () => { console.log(this); }

Ejercicio 4

¿Qué es el contexto de ejecución (this) en JavaScript?

this es una referencia al objeto que está ejecutando el código actual. Su valor depende de cómo se llama a la función:

  • En el ámbito global: this es el objeto global (window en navegadores, global en Node.js).
  • En un método de objeto: this es el objeto propietario del método.
  • En una función normal: this es el objeto global (en modo estricto, es undefined).
  • Con call(), apply(), bind(): this se puede establecer manualmente.

Ejercicio 5

¿Cómo se maneja el contexto de ejecución en arrow functions?

Las arrow functions no tienen su propio this. En lugar de eso, toman el this del ámbito donde fueron definidas (this léxico). Esto las hace ideales para callbacks dentro de objetos o clases, donde se quiere mantener el this del contexto padre.

Ejemplo
const persona = {
nombre: "Juan",
saludar: function() {
setTimeout(() => {
console.log("Hola " + this.nombre); // ✅ this es persona
}, 1000);
}
};

Ejercicio 6

¿Qué es el closure en JavaScript?

Un closure es la combinación de una función y el entorno léxico (el ámbito de variables donde fue declarada). Esto permite que una función interna acceda a variables de la función externa incluso después de que la función externa haya terminado de ejecutarse.

Ejemplo
function crearContador() {
let contador = 0; // variable del entorno léxico
return function() {
contador++; // closure: recuerda contador
return contador;
};
}
const incrementar = crearContador();
console.log(incrementar()); // 1
console.log(incrementar()); // 2

Entorno léxico: Es el ámbito de variables que rodea a una función en el momento de su definición. Incluye las variables locales de la función externa y las variables globales.

Ejercicio 7

¿Cuáles son las ventajas de usar callbacks?

  • Asincronía: Permiten ejecutar código después de que una operación asíncrona (como un setTimeout o una petición HTTP) termine, sin bloquear el hilo principal.
  • Reutilización: Se puede pasar la misma función callback a diferentes funciones para personalizar su comportamiento.
  • Modularidad: Separan la lógica principal de la lógica personalizada, haciendo el código más limpio.
  • Eventos: Son la base del manejo de eventos (clics, teclado, etc.) en el navegador.

Ejercicio 8

¿Qué es el callback hell y cómo se puede evitar?

El callback hell (también llamado "pirámide de la perdición") es un problema de legibilidad que ocurre cuando múltiples callbacks se anidan uno dentro de otro, creando una estructura difícil de leer y mantener.

Ejemplo de callback hell
setTimeout(() => {
setTimeout(() => {
setTimeout(() => {
console.log("demasiado anidado");
}, 1000);
}, 1000);
}, 1000);

Cómo evitarlo:

  • Usar Promesas (.then() encadenados)
  • Usar async/await (sintaxis más limpia)
  • Modularizar: extraer callbacks en funciones separadas con nombres descriptivos
  • Usar librerías como async.js para manejar flujos complejos

Ejercicio 9

¿Qué es la composición de funciones?

La composición de funciones es el proceso de combinar dos o más funciones para crear una nueva función, donde la salida de una función se convierte en la entrada de la siguiente. Es un concepto central de la programación funcional.

Ejemplo
const sumarUno = x => x + 1;
const duplicar = x => x * 2;
const sumarYDuplicar = x => duplicar(sumarUno(x));
console.log(sumarYDuplicar(5)); // (5+1)*2 = 12
// Con compose
const compose = (f, g) => x => f(g(x));
const procesar = compose(duplicar, sumarUno);
console.log(procesar(5)); // 12

Ejercicio 10

¿Qué es la currying y para qué se usa?

El currying es una técnica de la programación funcional que transforma una función que recibe múltiples parámetros en una secuencia de funciones, cada una recibe un solo parámetro y devuelve otra función hasta que todos los parámetros han sido proporcionados.

Ejemplo
// Función normal
function sumar(a, b, c) { return a + b + c; }
// Versión currificada
function sumarCurry(a) {
return function(b) {
return function(c) {
return a + b + c;
};
};
}
const sumar5 = sumarCurry(5);
const sumar5y3 = sumar5(3);
console.log(sumar5y3(2)); // 10
// En una línea con arrow functions
const suma = a => b => c => a + b + c;

Para qué se usa: Permite crear funciones parcialmente aplicadas, facilita la composición de funciones, mejora la reutilización y hace el código más declarativo.
