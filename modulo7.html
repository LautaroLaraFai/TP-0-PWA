Ejercicio 1

¿Qué significa que JavaScript sea un lenguaje asincrónico y no bloqueante?

Asincronía: Las tareas pueden ejecutarse en segundo plano mientras otras continúan. Esto permite que operaciones que toman tiempo (como leer un archivo o esperar una respuesta de red) no detengan la ejecución del resto del código.

No bloqueante: Las operaciones devuelven el control inmediatamente, independientemente de si la tarea ha terminado o no. El hilo de ejecución no se queda detenido esperando.

Ejemplo
console.log("1");
setTimeout(() => console.log("2"), 1000);
console.log("3");
// Salida: 1, 3, 2 (no bloquea)

Ejercicio 2

¿Qué es el Event Loop y qué rol cumple en la asincronía?

El Event Loop (bucle de eventos) es el mecanismo que permite a JavaScript manejar operaciones asíncronas. Su función es estar continuamente verificando si hay tareas pendientes en la cola de mensajes y ejecutarlas cuando el call stack está vacío.

Componentes principales:

  • Call Stack: Pila de ejecución de funciones.
  • Task Queue (Callback Queue): Cola donde se guardan los callbacks listos para ejecutarse.
  • Event Loop: Ciclo que transfiere tareas de la cola al call stack cuando está vacío.
Funcionamiento
1. Ejecuta código síncrono
2. Las tareas asíncronas (setTimeout, fetch) pasan a Web APIs
3. Cuando terminan, sus callbacks van a la Task Queue
4. Event Loop las pasa al Call Stack cuando está vacío

Ejercicio 3

¿Qué es una promesa (Promise) y cuáles son sus tres posibles estados?

Una Promise es un objeto que representa la eventual finalización (o falla) de una operación asíncrona. Es como un "contrato" que ahora no tiene valor, pero lo tendrá en el futuro.

Tres estados:

  • Pending (pendiente): Estado inicial, no se ha completado ni rechazado.
  • Fulfilled (resuelta): La operación se completó con éxito.
  • Rejected (rechazada): La operación falló.
Ejemplo
const promesa = new Promise((resolve, reject) => {
// estado: pending
if (todoOk) resolve("Éxito"); // estado: fulfilled
else reject("Error"); // estado: rejected
});

Ejercicio 4

¿Cómo se crea una promesa manualmente?

Se utiliza el constructor new Promise() que recibe una función ejecutora (executor) con dos parámetros: resolve y reject.

Ejemplo
function esperar(ms) {
return new Promise((resolve) => {
setTimeout(resolve, ms);
});
}
const promesaConError = new Promise((resolve, reject) => {
const exito = false;
if (exito) resolve("Ok");
else reject("Algo salió mal");
});

Ejercicio 5

¿Cómo se maneja el resultado de una promesa usando .then() y .catch()?

.then() se ejecuta cuando la promesa se resuelve (fulfilled) y recibe el valor resuelto. .catch() se ejecuta cuando la promesa se rechaza (rejected) y recibe el error.

Ejemplo
miPromesa
.then(resultado => {
console.log("Éxito:", resultado);
})
.catch(error => {
console.log("Error:", error);
});

Ejercicio 6

¿Qué ventaja tiene async/await frente a .then()?

async/await es azúcar sintáctico sobre las promesas. Permite escribir código asíncrono que se lee como si fuera síncrono, más legible y fácil de mantener. Evita el anidamiento excesivo y facilita el manejo de errores con try/catch.

Comparación
// Con .then()
fetch(url)
.then(res => res.json())
.then(data => console.log(data));
// Con async/await
const response = await fetch(url);
const data = await response.json();
console.log(data);

Ejercicio 7

¿Qué ocurre si dentro de una función async ocurre un error y no se usa try/catch?

Si una función async lanza un error y no se captura con try/catch dentro de la función, la promesa que devuelve se rechaza automáticamente con ese error. Quien consuma la función deberá manejarlo con .catch() o un try/catch externo.

Ejemplo
async function obtenerDatos() {
throw new Error("Fallo interno"); // la promesa se rechaza
}
obtenerDatos().catch(err => console.log(err)); // Se captura acá

Ejercicio 8

¿Qué hace Promise.all() y en qué casos puede fallar?

Promise.all() recibe un array de promesas y devuelve una nueva promesa que se resuelve cuando todas las promesas del array se resuelven. Devuelve un array con los resultados en el mismo orden.

Falla cuando: Cualquiera de las promesas del array es rechazada. En ese caso, Promise.all() se rechaza inmediatamente con el error de la primera promesa que falla.

Ejemplo
Promise.all([promesa1, promesa2, promesa3])
.then(resultados => console.log(resultados))
.catch(error => console.log("Una falló:", error));

Ejercicio 9

¿Cuál es la diferencia entre Promise.all() y Promise.race()?

Promise.all(): Espera a que todas las promesas se resuelvan. Si alguna falla, todo falla.

Promise.race(): Se resuelve (o rechaza) con el resultado de la primera promesa que se complete (sin importar si fue resuelta o rechazada).

Ejemplo
// Promise.race() devuelve la primera que termine
Promise.race([
new Promise(r => setTimeout(() => r("A"), 1000)),
new Promise(r => setTimeout(() => r("B"), 500))
]).then(resultado => console.log(resultado)); // "B" (la más rápida)

Ejercicio 10

¿Qué es una función async? ¿Qué devuelve siempre?

Una función async es una función declarada con la palabra clave async. Esta característica permite usar await dentro de ella para esperar promesas.

Lo que devuelve siempre: Una función async siempre devuelve una Promesa. Si el código retorna un valor que no es una promesa, JavaScript lo envuelve automáticamente en una promesa resuelta. Si ocurre un error, devuelve una promesa rechazada.

Ejemplo
async function saludar() {
return "Hola"; // automáticamente envuelto en Promise.resolve("Hola")
}
saludar().then(console.log); // "Hola"
async function fallar() {
throw new Error("Ups"); // devuelve Promise.reject
}
