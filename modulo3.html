Ejercicio 1

¿Cómo se declara una función en JavaScript? ¿Cuál es la diferencia con una arrow function? ¿Qué pasa si llamás a una función antes de declararla con function? ¿Y si es una arrow function?

Declaración: function nombre(parametros) { /* código */ }

Arrow function: const nombre = (parametros) => { /* código */ } o versión simplificada si es una sola línea.

Diferencia principal: Las arrow functions no tienen su propio this, no tienen arguments y no pueden usarse como constructores.

Llamada antes de declarar: Con function funciona por hoisting (se eleva la declaración). Con arrow function NO funciona, da error porque se asigna a una variable que aún no existe.

Ejemplo
saludar(); // ✅ funciona
function saludar() { console.log("Hola"); }
saludar2(); // ❌ error
const saludar2 = () => console.log("Hola");

Ejercicio 2

¿Qué diferencia hay entre una función que retorna un valor y una que no?

Una función que retorna un valor devuelve un resultado que puede ser usado fuera de ella. Una función que no retorna, implícitamente devuelve undefined y generalmente se usa para ejecutar acciones (modificar algo, mostrar en consola, etc.).

Ejemplo
function sumar(a, b) { return a + b; } // retorna valor
function saludar() { console.log("Hola"); } // no retorna (undefined)

Ejercicio 3

¿Qué es el scope? ¿Qué diferencia hay entre una variable global y una local?

Scope: Es el alcance o ámbito donde una variable es accesible.

Variable global: Se declara fuera de cualquier función o bloque y es accesible desde cualquier parte del código.

Variable local: Se declara dentro de una función o bloque y solo es accesible dentro de ese ámbito.

Ejemplo
let global = "accesible desde cualquier lugar";
function ejemplo() {
  let local = "solo dentro de la función";
}
console.log(local); // ❌ error, no existe fuera

Ejercicio 4

¿Qué pasa si una función accede a una variable que no fue declarada dentro de ella?

JavaScript busca la variable en los scopes superiores (scope chain). Si no la encuentra en ningún lado, con var se crea automáticamente como global (modo no estricto), pero con let o const da error de referencia.

Ejemplo
function ejemplo() {
  console.log(noDeclarada); // ❌ ReferenceError
}

Ejercicio 5

¿Qué es un closure? ¿Podés explicarlo con un ejemplo simple?

Closure: Es una función que "recuerda" el ámbito en el que fue creada, incluso después de que ese ámbito haya terminado de ejecutarse.

Ejemplo
function crearContador() {
  let contador = 0;
  return function() {
    contador++;
    return contador;
  };
}
const incrementar = crearContador();
console.log(incrementar()); // 1
console.log(incrementar()); // 2

Ejercicio 6

¿Qué pasa si una función se llama a sí misma? ¿Cómo se llama eso?

Se llama recursión. La función se invoca a sí misma hasta que se cumple una condición de corte.

Ejemplo
function factorial(n) {
  if (n === 0) return 1;
  return n * factorial(n - 1); // se llama a sí misma
}
console.log(factorial(5)); // 120

¿Qué pasa si una función tiene múltiples parámetros (4 por ejemplo) pero se llama con menos (3)?

No da error. Los parámetros faltantes toman el valor undefined. Se pueden asignar valores por defecto para evitarlo.

ejemplo(1, 2, 3); // 1, 2, 3, undefined

Ejercicio 8

¿Qué diferencia hay entre declarar una función dentro de otra, o fuera?

Una función declarada dentro de otra solo es accesible dentro de esa función (privada). Una función declarada fuera es accesible globalmente. Se usa cuando la función interna es auxiliar y no debe ser usada fuera.

Ejemplo
function externa() {
  function interna() { console.log("solo dentro"); }
  interna(); // ✅ funciona
}
interna(); // ❌ error, no existe fuera

Ejercicio 9

¿Se puede guardar una función en una variable? ¿Para qué sirve eso?

Sí, en JavaScript pueden asignarse a variables, pasarse como argumentos y retornarse desde otras funciones.

Ejemplo
const saludar = function() { console.log("Hola"); };
const operacion = (a, b) => a + b;
function ejecutar(fn, valor) { return fn(valor); } // función como parámetro

Utilidad: Permite programación funcional, callbacks, funciones de orden superior, y más flexibilidad en el código.
